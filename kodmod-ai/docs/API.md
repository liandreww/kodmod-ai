# KODMOD AI — API Reference

Base path: `/api/v1` (configurable via `API_PREFIX`).
Auth: `Authorization: Bearer <jwt>` for protected routes.
Errors: standard HTTP codes; body is `{ "detail": "..." }`.

OpenAPI is auto-generated by FastAPI and served at `/docs` and `/redoc`.

---

## Voice

### `POST /voice/chat` — single-turn audio
Accepts a multipart upload `audio` (wav/mp3/ogg, ≤ 120 s).
Returns the final spoken answer URL plus the transcript.

### `POST /voice/text` — text fallback
Body: `VoiceChatRequest` (`student_id`, `text`, optional `session_id`).
Returns `VoiceChatResponse`.

### `WS /ws/voice?token=<jwt>` — bidirectional streaming
Client → server frames:
- `binary`: raw 16 kHz PCM little-endian mono chunks.
- `text` JSON `{"event": "end_of_utterance"}`: explicit utterance boundary.

Server → client frames:
- `text` JSON `{"event": "partial", "text": "..."}` — partial transcript.
- `text` JSON `{"event": "final_transcript", "text": "..."}`.
- `text` JSON `{"event": "tutor_token", "text": "..."}` — streaming tokens.
- `binary`: TTS audio frames.
- `text` JSON `{"event": "turn_end"}`.

## Quiz

### `POST /quiz/start` → `QuizStartResponse`
Begins a new quiz session. Returns `quiz_session_id` + the first question.

### `POST /quiz/submit` → `QuizSubmitResponse`
Submits an answer. Resumes the graph from the saved checkpoint, runs
scoring + (if last) analyzer, returns either the next question or the
final summary.

## Student

### `GET /student/me` → `StudentOut`
Current student (from JWT).

### `POST /student` → `StudentOut`
Create a student (admin / onboarding).

### `GET /student/{id}/profile` → `StudentProfileOut`
Extended profile including overall mastery, weak/strong concepts, streak.

---

## Analytics

### `GET /analytics/student/{id}?window={today|week|month|all}`
Per-student rollup.

### `GET /analytics/student/{id}/spoken?window=...`
Adds an audio-friendly summary string for the dashboard's "play" button.

### `GET /analytics/classroom/{id}?window=...`
Teacher view: per-class stats + weakest concepts + per-student summaries.

### `GET /analytics/classroom/{id}/alerts`
Just the alert list — for a notification badge on the teacher dashboard.

---

## Content

### `GET /content/concepts?subject_id=<uuid>`
List concepts (optionally filtered by subject).

### `GET /content/concepts/{id}`
Concept details.

### `GET /content/concepts/{id}/lessons`
Lessons attached to a concept.

### `POST /content/retrieve`
Direct RAG retrieval — primarily for debugging and teacher tools.
Body: `ContentRetrieveRequest`.

---

## Exercise

### `POST /exercise/generate` → `ExerciseGenerateResponse`
Adaptive question batch (uses Problem Generator under the hood).

### `GET /exercise/by-concept/{id}` → `list[ExerciseOut]`
Static teacher-authored exercises filtered to audio-friendly ones.

- `GET /health/version` — runtime version metadata
